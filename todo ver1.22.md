# TODO ver1.22

## 現状の事象
- Fish系で allowEmpty=true かつ rod default/rare を null にした場合でも、ランタイムでは「元の種族（バニラ/RHHの釣りテーブル）」が返ってくる挙動になっている。結果として、Route 102 Night Super Rod でヘイガニ（バニラ設定）が釣れるなど、遭遇ゼロにはならない。

## 理想の挙動
- allowEmpty=true で WL が空のときは、「元種族を返さず、遭遇なし（釣れない）」にする。釣り・野生・トレーナー共通で空許容時は出現をスキップする挙動に統一。
- allowEmpty=true は「このスコープでは出現させない」ことを厳格に示すフラグとして扱い、効果範囲が強い（意図せずバニラ種が残らない）ことを明記する。

## 修正方針（案）
- ランタイム（src/randomizer.c）
  - GetRulePools で wlCount==0 かつ allowEmpty の場合、RandomizeWildEncounter/Trainer/Fixedいずれも「スロット0扱い」で遭遇なしにする（SPECIES_NONEは使わず、処理側で遭遇スキップ）。釣りはバイト成功後に専用メッセージで不発扱い。
  - 遭遇が発生しないため通常ログは出ないが、allowEmptyでスキップした事実をWARNで出す（フック時に1回）。釣りも同様に不発判定時にWARN。
  - トレーナー/固定は、スキップ時に手持ち枠が欠けないよう同じポケモン（従来の種か候補の先頭など）で埋める運用にする。将来変更余地は残すが、現段階では手持ち枠を減らさない方針。
- 生成スクリプト
  - allowEmpty=true で空を明示する場合以外でWLをnullのままにしたらビルドエラー（allowEmpty=falseでnullはNG）。nullを置く場合は必ずallowEmpty=trueで意図を明示させる。野生/釣り/トレーナー共通。
- 釣りのメッセージ（新規）:
  - allowEmpty=true かつ空WLで釣り処理が呼ばれた場合、「No Pokémon can be hooked here.」のような英語メッセージを新規に用意して表示する（既存のバイト失敗文言と区別する）。
  - 判定タイミングは「釣り上げる前」に行い、バイト成功後ではなく、リール直後に不発扱い＋メッセージ表示。
  - 表示箇所: 釣りミニゲーム開始直後（Old/Good/Super Rod使用直後・フィールド画面に切り替わった瞬間）。SELECTに登録している場合もSELECT押下直後に同様の判定でメッセージを出し、ミニゲームに入らない。
