## INCGFX Migration Impact

1.15.2 の最も大きい変更は asset pipeline。従来 docs にある `INCBIN_U8("...4bpp")` / `INCBIN_U16("...gbapal")` 前提は、1.15.2 後は一部で古くなる。

確認した代表例:

| 1.15.1-style idea | 1.15.2-style idea |
|---|---|
| generated `.4bpp` / `.gbapal` を `INCBIN_*` で読む | source `.png` / `.pal` と format string を `INCGFX_U8` / `INCGFX_U16` / `INCGFX_COMP` に渡す |
| `graphics_file_rules.mk` / `spritesheet_rules.mk` に多数の asset rule | `preproc` / `scaninc` が source から generated assets を追跡 |
| asset 追加 docs は generated binary file path を前提にしがち | 1.15.2 後は source image path と `INCGFX_*` declaration の両方を確認 |

Upgrade 時の注意:

- upstream changelog は clean worktree から `migration_scripts/1.15/migrate_incgfx.py` を使うことを示している。
- 既に独自 graphics を追加している場合、merge 前に `INCBIN_*` declarations を一覧化する。
- 新規 UI、Pokemon icon、battle UI、DexNav UI、move relearner UI の asset docs は 1.15.2 merge 後に必ず再確認する。
- `Makefile` は `ASSETS_DIR_NAME := $(BUILD_DIR)/assets` を導入しており、generated asset の出力 / clean 対象が変わる。

## Upgrade Recommendation

main を 1.15.2 へ上げる場合の安全な順序:

1. 現在の local customization を commit 済みにする。docs-only PR と source change PR は分ける。
2. clean worktree で `RHH` tag / changelog / compare を確認する。
3. 1.15.2 merge branch を切り、`git pull RHH expansion/1.15.2` または tag merge を行う。
4. INCGFX migration conflict を最初に解決する。asset pipeline が変わるため、UI / Pokemon graphics の独自差分がある場合はここが最優先。
5. build 後に `make clean` 相当の generated asset cleanup を含めて再 build する。
6. `docs/upgrades/upstream_diff_checklist.md` の 1.15.2 section を使って、battle / DexNav / map script / SaveBlock3 / icon / move relearner を再確認する。
7. docs の baseline 表記を 1.15.2 へ更新するか、1.15.1 調査 docs と 1.15.2 追記 docs を併存させるか決める。
