# TODO ver1.30（設計メモ）  

## 目的  
- エリアルール未定義の場所では、野生遭遇（陸/水/岩/釣り/隠し）を発生させない。  
- 釣りも同様にスキップ（ミニゲームに入る前に不発扱い）。  
- トレーナー/固定/アイテムなどは従来どおり（エリアルールなしでもランダム化を継続）。  

## 現状の挙動（v1.22まで）  
- `FindAreaRule(...)` が `NULL` の場合、従来のグローバルランダム化にフォールバックする。  
- allowEmpty+空WLのときのみスキップ（釣りはメッセージ表示、それ以外は遭遇なし）。  
- 釣りは allowEmpty でメッセージ “No POKéMON can be hooked here.” を表示。  

## 変更方針（設計案）  
### 野生（陸/水/岩/隠し/釣り）  
- `RandomizerAreaRulesEnabled()` かつ `FindAreaRule(...) == NULL` のとき、遭遇をスキップ。  
- ログ: WARNで「ルールなし → スキップ」を1回出す（例: `[WARN] RandR no rule map=G/N mask=X (skip)`）。  
- 釣り: ミニゲーム開始前に同様判定でスキップ。メッセージは「ここでは釣っても意味がない」趣旨の新規文言を用意（allowEmpty用と区別）。  
- タイムスロット: ルール未定義の場合もスキップを優先（ANY/時間帯問わず）。  
- 結果として「許可（ルール定義）したエリア以外では野生ポケモンが出現しない」仕様にする。  
- 出現対象はWLに列挙された種のみ（WL外は登場しない）。ブラックリスト/allowLegendOverrideの扱いは従来どおり。  
- フォームは「別種扱い」。WLに列挙がなければ登場しない（未指定フォームが出るのはデータ側のミスとみなす）。  
- 伝説カテゴリの扱い: デフォルトは出現不可。出したい場合は WL に明示し、`specialOverrides` で対応カテゴリを true にする（未指定なら出ない）。  
- 種別カテゴリの明示（例）: 伝説/準伝説/幻/ウルトラビースト/パラドックス等を定義し、「伝説」と呼ぶときはこれら全てを含むか、伝説のみを指すかを明確に切り分ける。`specialOverrides` でカテゴリごとに解禁する前提。カテゴリ別に緩和したい場合は個別フラグ（legend/mythical/ub/paradox/subLegend など）を使用。  
- カテゴリ切り分けにより、運用上「パラドックスのみ登場」「伝説のみ登場」等の設計が可能になる（カテゴリごとのフラグ/設定を将来追加して拡張する余地）。  
- カテゴリ草案（例）:  
  - 伝説: ミュウツー/ルギア/ホウオウ/パルキア/ディアルガ/ギラティナ/レシラム/ゼクロム/キュレム/ゼルネアス/イベルタル/ソルガレオ/ルナアーラ/ザシアン/ザマゼンタ/ムゲンダイナ/コライドン/ミライドン 等  
  - 準伝説: 三鳥/三犬/三闘/湖の三柱/レジ系/ラティ兄妹/クレセリア/コバルオン系/ランドロス系など  
  - 幻: ミュウ/セレビィ/ジラーチ/デオキシス全フォーム/ダークライ/シェイミ/マナフィ/アルセウス/ビクティニ/メロエッタ/ゲノセクト/ケルディオ/フーパ/ディアンシー/マギアナ/マーシャドー/ザルード/メルタン/メルメタル 等  
  - UB: テッカグヤ/デンジュモク/マッシブーン/フェローチェ/カミツルギ/アクジキング/ツンデツンデ/ズガドーン 等  
  - パラドックス: 現世/未来種（イダイナキバ/ハバタクカミ/テツノブジン 等）  
  - 一般枠に含める例: ジガルデ10%（50%は伝説扱い）、コスモッグ/コスモウム、フィオネ、ダクマ、メルタン。スワームチェンジは一旦許容。  
  - 準伝説枠に含める例: ウーラオス（全スタイル）、オーガポン、メルメタル。進化方法は別途調整。  

### トレーナー/固定  
- ルール未定義でも従来どおりランダム化を行う（スキップしない）。  
- 将来、エリア未定義でスキップしたい場合に備えて、別フラグで切替可能にする余地を残す。  

### 設定切替の検討

- オプション（必要なら）: グローバルランダム化を許可するconfigフラグを追加検討（デフォルトはOFF）。

## 懸念/検討ポイント  
- 釣りメッセージ: 未定義エリアスキップ用に「ここでは釣っても意味がない」新規文言を追加し、allowEmpty用と区別する。  
- エリア未定義をスキップする運用がデータ側の想定に反しないか（必ずルールを定義する運用が前提なら問題なし）。  
- トレーナー/固定をスキップしない方針を明記（野生だけ対象）。  
- ログ量とレベル: WARNで出すが、同一マップ/areaMask/timeSlot/釣りフラグの組み合わせでは1回に抑制し、マップ切替時にリセットするレートリミットを検討（デバッグビルド限定）。  
- デバッグ用バニラ遭遇トグル: ルール未定義でも「バニラ遭遇を一時許可」するデバッグフラグ＋例外マップリストで切替を検討（全許可と個別許可の2段階運用を想定）。  
- デバッグ用バニラ遭遇トグルの有効範囲: デフォルトは全種別（陸/水/岩/釣り/隠し）を一括でバニラ許可する単一フラグとし、カテゴリフィルタは無効化しない。必要になれば種別ごとの細分化フラグを追加する方向で検討（初期実装ではグローバルのみ）。  
- タイムスロット強制スキップ時のログ粒度: 未定義エリアでのスキップは時間帯に関係なく出力（時間帯ごとに抑制せず、マップ基準で抑制）。  
- エンカ率ゼロと遭遇ゼロの区別: エンカ率0マップ（室内等）と「未定義スキップ」をログやデバッグ出力で判別できるようにする。可能なら遭遇データのエンカウント率も併記するとデバッグしやすい。  
- 釣りメッセージの位置: ミニゲーム前表示で確定（allowEmptyスキップと同じタイミング）。  
- カテゴリフラグとWL整合: WLに伝説があって `specialOverrides.legend=false` の場合は登場しない。該当カテゴリ以外の候補が無ければ遭遇ゼロになる（警告を出すか要検討）。  
- 他カテゴリ（mythical/ub/paradox/subLegend 等）も同様に、WLに該当種があってフラグが false なら出現せず、候補が無ければ遭遇ゼロ（警告検討）。  
- 警告出力: 可能なら生成スクリプト（python3）側で警告を出し、難しければデバッグログで通知する方針。  
- フォーム漏れ検知: ベース種がWLにあるのにフォームがゼロの場合は警告（意図的にそうする可能性もあるためエラーではなく警告扱い）。  
- 警告抑制オプション: 理想は警告を無視するオプションを設けるが管理が煩雑になるため、当面は警告を出し続ける方針（抑制フラグは見送り）。  
- 釣りメッセージ未設定: 未定義エリアスキップ用メッセージが未設定ならエラーとする（共通メッセージを強制）。  
- 例外マップ/デバッグトグル: デバッグフラグでバニラ遭遇を許可した場合でも、例外マップで有効になっていることをデバッグログで明示（どのマップでバニラ許可されたか分かるようにする）。  
- UIメッセージの多言語対応: 日本語化は行わず英語のみで運用（GBA版メッセージも英語固定で追加）。  
- Rod/釣りの挙動: 未定義エリアもスキップ対象。ミニゲーム前に判定し、新規メッセージで終了（allowEmptyと区別）。specialOverridesのカテゴリフィルタも釣りWLに適用。未定義スキップはWARNを1回（マップ基準レート制御）、例外マップ/デバッグトグルも釣りに適用。  
- 未定義エリアスキップのUI: 何も起きない（ログのみ）でOK。WLで特定エリアに許可を出す運用前提。  
- カテゴリ重複時の扱い: 重複する種（例: タケルライコ＝準伝＋パラドックス）は論理OR（どれかtrueなら出現可）を基本とし、厳格ANDに切替できる設定（フラグ）を用意する案を検討。  
- 重複カテゴリの論理切替: OR/AND を切り替えるグローバルフラグ（例: `specialOverridesMode = OR|AND`）を用意する案を検討。デフォルトOR、ANDは厳格運用向け。  
- 重複カテゴリ論理は全マップ共通で適用（エリア個別切替は行わない）としてバランス調整をシンプル化する。  
- カテゴリ判定ソース: 種→カテゴリのマッピングは既存定数（例: `include/constants/species.h`）から派生させ、スクリプトで自動利用できる形を目指す（手書きリストを減らす）。  
- 未分類カテゴリ検出の閾値: 未分類種は1件でも警告。件数が多い場合はサマリを出すが、ビルドは継続（致命エラーにはしない）。  
- エラーメッセージ粒度: ビルド時警告/エラーはマップ・areaMask・timeSlot・カテゴリ名・不足/無効の種数を表示し、警告が多い場合でも原因を特定しやすくする。  
- 設定フラグの配置: グローバル切替（例: specialOverridesMode OR/AND、デバッグ用バニラ許可など）は config/ヘッダの定数やflagに集約し、データ(YAML)側に混在させない。配置先は既存の config/define 系（`include/constants/flags.h`/`config.h` など）にまとめる。  
- 設定フラグの具体配置:  
  - ビルド時固定値（例: specialOverridesModeデフォルトOR/AND）は `config.h` で定数化。  
  - ランタイム切替フラグ（例: バニラ遭遇許可、デバッグログON/OFF）は `include/constants/flags.h` に新規FLAGを置く方針。  
  - グローバル変数を避け、既存のflag/var枠を優先利用（空きが無ければ新規定義）。  
  - ランダマイザ専用の定数は `include/constants/randomizer.h` 等にまとめる案も検討（defineでモードを一元管理）。  
- テキストIDの固定: 未定義釣りスキップ用メッセージは strings.h/c に専用IDを追加（例: `gText_NoFishingTargets`）し、スクリプト/ランタイムでこのIDに固定する。  
- テストサンプル: 未定義エリア＝遭遇なし、定義あり＝遭遇あり、allowEmptyスキップ（陸/釣り両方）、カテゴリフィルタ（legend/mythical/ub/paradox/subLegendのON/OFF）、重複カテゴリ（タケルライコ等でOR/AND切替）、釣りメッセージ表示タイミング、例外マップON/OFF、バニラトグルON/OFF、specialOverridesMode OR/AND、allowEmpty+rareの組み合わせをカバーする簡易YAMLを用意し、生成スクリプトで自動チェックする。  
- スクリプト警告の優先順位: 未定義エリアスキップ > メッセージ未設定 > カテゴリ/WL不整合 > フォーム漏れ > その他 の順で表示を検討（原因特定を優先）。  
- reduce/merge重複: キットの加減算後にカテゴリフィルタが適用されることを明記（キットで伝説を追加してもspecialOverridesがfalseなら出ない）。生成スクリプトで重複種の警告やソート有無も検討。  
- 釣り以外のUI変化: 未定義スキップ時は何も出さず、SE/フィールドエフェクト等の副作用が残らないよう留意（釣り以外はメッセージなしでOK）。  
- OR/AND グローバル切替: デフォルトAND、オプションでORに切替可能とする（デバッグ/設定で変更）。  
- YAMLスキーマバージョン管理: 旧スキーマはエラー扱い（明示必須化）。徹底してビルドエラーを発生させる方針で移行を強制する。  
- スキーマversionキーを必須化: `schemaVersion: 1.30` のように先頭で明示し、未指定/未知値は即ビルドエラー。スキーマ更新時は値を上げ、旧値はサポートしない。将来値は「既知のキー集合＋各キーの型が完全一致する場合のみ受理（pythonスキーマ検証が通る形以外は不可）」とし、それ以外はエラー。  
- セーブ互換メモ: 未定義エリアにいるセーブで再開した場合、遭遇がなくなる挙動を仕様として周知（挙動変化を認識させる）。  
- カテゴリマッピング更新フロー: 新種追加時に `include/constants/species.h` から派生テーブルを更新し忘れないよう、未分類種があればスクリプトで警告。  
- 例外マップリストの配置とスコープ: デバッグ専用ファイルに例外リストを置き、本番ビルドでは無効化（誤運用防止）。  
- 例外マップリストの本番無効化: `config.h` 等で本番時は強制OFFにするスイッチを用意し、デバッグビルドでのみ有効にする（誤運用防止）。  
- 警告スパム対策: 警告は全件出す方針。サマリ表示で量を把握し、閾値でビルド失敗に昇格する運用は当面採用しない（必要になれば再検討）。  
- テストカバレッジ: OR/AND切替、カテゴリフィルタON/OFF、未定義スキップ＋釣りメッセージ、例外マップON/OFF、allowEmpty など主要パスを自動チェックするサンプルを用意。  
- 警告スパムの具体例と対策: カテゴリ不整合多数/フォーム漏れ多数/未分類種多数/未定義エリア大量などで警告が爆発するケースを想定。全件出力＋サマリ表示で把握し、ビルド失敗への昇格は当面見送り（必要時に再検討）。  
- 未定義エリア大量の扱い: 初期開発段階で未定義エリアが多くてもビルドエラーにはしない。警告のみで認知させ、後でルールを埋める前提。  
- 例外マップリストの適用範囲: 例外マップは野生全般（陸/水/岩/釣り/隠し）に適用し、スキップ判定を無効化してバニラ遭遇を許可する。  
- ログ抑制キー: WARN抑制は mapGroup/mapNum/areaMask/timeSlot/釣りフラグを含めたキーで制御し、時間帯・エリア種別を跨ぐ場合は再度出力する。  
- カテゴリ警告の閾値: 上限は設定せず、警告はすべて出す方針。閾値昇格は行わない（多すぎる場合は個別相談で方針を見直す）。  

## 決定事項（カテゴリフラグ関連）  
- メッセージ: 未定義エリアスキップ用は新規文言（釣りも同様に新規文言）。  
- グローバルランダム化の許容フラグは当面なし（デフォルトは「ルールなし→スキップ」）。  
- カテゴリ拡張フラグ: `specialOverrides` で legend/mythical/ub/paradox/subLegend 等を加算的に解禁（複数ONで合算解禁）。  
- デフォルト値はすべてfalse。trueにしたカテゴリのみ出現可。エリア単位でネスト指定（例: `specialOverrides: { legend:true, subLegend:true, mythical:true, ub:true, paradox:true }`）を基本とし、単一の旧フラグは廃止（併記はエラー/警告想定）。  
- 位置づけ補足: WL（キット等）で種を列挙した上で、`specialOverrides` はその中から該当カテゴリを解禁するフィルタとして使う。
